# A-Share Research Skills

一个围绕 **A 股超跌分析、阶梯建仓、历史回测** 打造的个人 Skill 仓库。  
目标不是堆功能，而是把一套常见但容易做歪的投资研究流程拆清楚、写明白、做成可复用模块。

这套仓库聚焦 3 件事：

- **先判断为什么跌**
- **再决定值不值得分批买**
- **最后用历史数据验证规则**

适合想把主观感觉，逐步变成结构化研究流程的人。

---

## Features

- **超跌排雷分析**  
  判断一只股票是错杀、周期承压，还是基本面恶化 / 爆雷

- **阶梯建仓方案设计**  
  基于真实波动和预算，输出 4 层买点、资金分配、止盈位、失效条件

- **真实历史回测**  
  用真实 A 股历史 K 线做机械回放，输出交易日志和结果汇总

- **明确边界，不装神**  
  不把跌幅当价值，不把回测当保证，不把框架包装成喊单

## Included Skills

### 1. `dianjin-analysis`
**点金术基础排雷 / 超跌分析**

用于回答最关键的问题：

> 这只股票跌这么多，到底是机会，还是坑？

它会先判断下跌原因，再给出结构化结论，而不是看到跌幅大就直接说“便宜了”。

**主要能力：**

- 识别下跌主因
- 区分：
  - 行业系统性下跌
  - 公司阶段性承压
  - 基本面恶化
  - 事件型暴跌 / 爆雷
- 检查营收、利润、现金流、资产负债、治理风险
- 给出最终判断：
  - **可看**
  - **谨慎**
  - **不碰**

**适合场景：**

- “这票是不是错杀？”
- “还能不能看？”
- “值不值得进观察池？”
- “帮我做点金术分析”

---

### 2. `dianjin-ladder`
**点金术阶梯建仓**

这个 Skill 不负责“证明股票值得买”，只负责：

> 在已经通过基础排雷之后，怎么分层建仓更合理？

它会基于真实历史波动和资金预算，设计更可执行的建仓方案，而不是拍脑袋写几个价格。

**主要能力：**

- 生成 4 层建仓点位
- 设计预算分配方案
- 支持常见阵型：
  - `1-2-3-4` 金字塔
  - `3-2-2-3` 稳健白马型
- 计算：
  - 各层买点
  - 各层预算
  - 各层止盈价
  - 分层完成后的测算持仓均价
- 给出失效条件，防止机械补仓失控

**适合场景：**

- “给我做个阶梯建仓方案”
- “20 万预算怎么分 4 层买？”
- “这只票适不适合 1-2-3-4 建仓？”
- “帮我算一个点金术补仓计划”

---

### 3. `dianjin-backtest`
**点金术阶梯建仓回测**

当你已经有一套规则，下一步就不是靠嘴说逻辑，而是：

> 把规则放进真实历史数据里跑一遍。

这个 Skill 会把建仓方案放到历史行情中做机械执行，输出完整回测记录。

**主要能力：**

- 抓取真实 A 股历史日 K 数据
- 生成操作手册
- 严格按历史行情回放买卖
- 处理真实交易细节：
  - 跳空成交
  - 佣金
  - 印花税
  - 滑点
  - A 股整手约束
- 输出结果文件：
  - `manual.json`
  - `trades.csv`
  - `summary.json`

**适合场景：**

- “把这套点金术规则做回测”
- “从某天开始回放这只股票”
- “我要看交易明细和收益汇总”
- “验证这个分层买法历史上表现怎么样”

## Recommended Workflow

建议按这个顺序使用：

### Step 1. 先排雷
使用 `dianjin-analysis`

先确认这只股票值不值得继续研究。  
如果属于基本面恶化或爆雷型，不进入后续流程。

### Step 2. 再做建仓
使用 `dianjin-ladder`

对通过初筛的标的，设计分层买点和资金分配。

### Step 3. 最后做回测
使用 `dianjin-backtest`

把规则放到历史数据中机械执行，验证这套方案过去的表现。

## Core Philosophy

这个仓库遵循几条很硬的原则：

- **先问为什么跌，再问能不能买**
- **先排雷，再做仓位管理**
- **不用假数据，不伪造成功**
- **不把技术反弹当基本面反转**
- **不把历史回测当未来承诺**
- **不把研究框架包装成投顾喊单**

## Notes

- 本仓库主要用于 **研究辅助与流程化分析**
- 不构成投资建议
- 回测结果仅代表历史条件下的机械执行结果
- 真实市场会变化，任何规则都可能失效
- 如果标的基本面已经坏掉，再漂亮的建仓模型也没用
